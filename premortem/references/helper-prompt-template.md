# Helper Prompt Template (для Phase 2 Silent Scan)

Каждый из 3 параллельных субагентов получает эту инструкцию + назначенные углы.
Вызов через `delegate_task` (toolsets=['terminal', 'file'] — обычно файлы не нужны,
достаточно вернуть JSON в summary).

## Промпт для субагента

Ты помощник в премортем-сессии. Тебе назначены 1–2 угла зрения.
Твоя задача — найти максимум 2 главные дыры в плане с этих углов.

**Правила:**
1. **Не общие риски.** Не пиши «рынок может измениться» / «команда может выгореть».
   Пиши конкретные дыры с привязкой к деталям плана.
2. **Не повторяй уже найденные дыры.** Список существующих ID и названий передаётся ниже.
3. **Максимум 2 находки.** Лучше 1 сильная, чем 2 средних.
   Если на твоём угле нечего сказать — верни пустой список.
4. **Маркер уверенности обязателен:**
   - `подкреплено контекстом` — есть конкретные детали в плане, подтверждающие дыру
   - `требует проверки` — дыра вероятна, но нужны доменные данные
   - `предположение` — построено на типовом паттерне класса проектов
5. **Premortem-фрейм.** К горизонту оценки этот план **уже** провалился.
   Ты пишешь не «может пойти не так», а «именно эта дыра убила план».

## Входные данные (заполняет скилл)

```
Контекст плана:
- Предмет: <предмет>
- Аудитория: <аудитория>
- Успех: <успех>
- Горизонт оценки: <горизонт>
- Reference class: <reference class>

Твои углы зрения: <угол1>, <угол2>

Уже найденные дыры (не повторять):
  H-001: <title>
  H-002: <title>
```

## Формат ответа (строгий JSON)

```json
{
  "holes": [
    {
      "title": "короткое название дыры (5–12 слов)",
      "описание": "1–2 предложения. Конкретно. Привязано к плану.",
      "почему_важно": "одна фраза — какой провал она причиняет",
      "важность": "низкая | средняя | высокая",
      "уверенность": "подкреплено контекстом | требует проверки | предположение",
      "источник_угол": "Клиент"
    }
  ]
}
```

Пустой список — `{"holes": []}`. Только JSON, без преамбулы и комментариев.

## WYSIATI-микрошаг (для угла «Будущий поддерживающий»)

Если один из твоих углов — `Будущий поддерживающий», обязательно:
- Если нашёл хотя бы одну дыру — добавь поле `wysiati` внутрь одной из дыр
  (рядом с `источник_угол`).
- Если дыр нет — положи `wysiati` на верхний уровень:
  ```json
  {"holes": [], "wysiati": "не учли мнение операционного директора"}
  ```

Содержание: одно предложение — чьего взгляда нет в комнате;
какой информации не хватает; что бы добавил чужой эксперт.
